 Visualization

The visualization is populated as you select and deselect tags. The nodes represent tags (Researchers, Departments, and Interests), and the lines the connection between them. The visualization supports the following actions:

Show Node Title Mouseover a node to reveal its title.

Get Node Detail Click on a node opens the node's information panel.

Expand Node Double-click to on a node reveal (expand) its connections.

Move Nodes Mouse click, hold and drag a node. The layout adjusts according to the acting forces.

Remove Node Right-click on a node to remove it from the visualization. All connected nodes are affected by this action.

Tags

The Tags Section lists (in alphabetical order) all tags available divided by type: Researcher, Interests, Department. Selected tags are marked with label [an orange tag icon]. Tags that are in relation to selected tags but not directed selected are visible in the visualization and marked in the list with label[a dark tag icon]. This section supports the following actions:

Select Click on a tag to add it (and its relations) to the visualization.

Deselect To deselect and remove a tag (and the relations currently connected only to this tag) from the visualization, click on an already selected tag.

Visible tags Click on a visible tag (marked with label[a dark tag icon])to remove from the visualization. Click again to select and add it (and all its relations) to the visualization.

Tag's details Click on info_outline[ info icon].

Filter Click on filter_list[the filter icon] at the top to filter the list to show only the currently selected and visible tags.

Search Click on search[the search icon] at the top to show the search field. The autocomplete component shows a few available options as soon as you start typing. Continue refining the term until you find it. The search engine looks in all the three types at the same type, so you might need to change tabs to find what you are looking for.

Clear selection Click on clear[the "X" icon] at the top to reset the selection in the visualization.

Layout

Here you can adjust, add and remove visual elements on the visualization.

Layout: General

Display Choose between network [default] and cluster. The network presents a set of forces (gravity, charge, and distance) acting on the nodes and links to show the relationship between them. The cluster display only shows nodes grouped according to an attribute set by the user and affected only by gravity.

Cluster (cluster display only) Choose the attribute to be used in the clustering process: type (see Tags) [default] or community (see below).

Gravity This force act on the nodes. The default devalue is 1. Lower values lose the tension and pull the nodes to the edges (and outside) of the screen. Higher values increase the tension and push the nodes to the centre.

Charge (network display only) This force adds/increases the "electric charge" of a node; that is, it attraction or repellent force. The default is -60. Lower values increase the charge and make the nodes repel themselves with more force. Higher values have the opposite effect.

Distance (network display only) This force acts directly on the links. The default value is 40. Lower values decrease the link's length, pushing the nodes closer to its respective source; higher values increase the length, pushing nodes away from the source.

Collision Detection When turned on, this option detects and resolves nodes overlapping. It requires more computational power and can make get slower if the visualization has a high number of nodes. It is off by default.

Community Community is not an option, but an attribute dynamically computed every time a tag is added or removed from the visualization. The algorithm tries to put together tags that have connections in common, creating what is called modularity class.

* You can use a combination of these options to fit the visualization on the screen size.

Layout: Nodes

Show nodes Show and hide nodes.

Size Use an attribute to size the nodes. The options are default or weight. While the default sets all nodes with the same size, weight (see below) sets the maximum size based on the node with the highest connections and the minimum size based on the node with the lowest number of connections.

Scale Works as a multiplier for the nodes' current size.

Colour There are three options: default, type (see tags), and community (see Display). The default set all nodes to black. Type colour code nodes by type in using the following colours: Blue (Researchers), Orange (Departments), and dark grey (Interests). The community option uses a colour palette to colourize each cluster randomly. These colours might change if you chose the dark theme (see Extra).

Weight Weight is not an option, but an attribute dynamically computed every time a tag is added or removed from the visualization. It is based on the number of connections currently shown in the visualization. For instance, if a node has 4 connections, its weight is 4.

Layout: Titles

Show titles Show and hide nodes' title.

Scale If turned on, titles inherent the nodes' scale (see Nodes).

Inherent colour If turned on, titles inherent the nodes' colour (see Nodes).

Layout: Links

Show links Show and hide the connections between the nodes.

Thickness Defines the stroke width. Choose between default and weight (see Nodes). While the default sets thickness to 1 for all links, weight sets the line thickens based on the weight of its source. Thus, if a source node has weight 4 and the target weight 1, the line that connects the two have the thickness 4.

Colour Choose between default and community (see Display). The default sets link’s colour to light grey. Community sets line colour based on the community of its source.

Strength This option sets the tension force in each link. There are two options min [default] and max. Min corresponds to the minimum value between a source and a target, making the tension stronger and the nodes closer to their source. Max corresponds to the maximum value between a source and a target, making the tension weaker and their nodes distance from their sources.

Layout: Extra

Dark theme Apply the dark theme: background turns black, and the default colours are inverted (see Nodes, Titles, and Links).

Gooey FX Makes the nodes appear liquid and deformed, blending one into the other. It is computationally heavy when applied to the visualization with a high number of nodes, but it might have a weak effect when only a few modes are present in the visualization. It is not fully exportable (see Export).

Import

Only editors and administrators can import data.

The Import section allows editors to import datasets of collections of relations between Researchers, Research Topics, and Departments. There are two options: JSON and CSV.

Import JSON

The JSON format can hold a list of items, their metadata, and a list of their relations. Use the following schema to import a JSON file.


[
 {
  "name": "String: required",
  "type": "String: required ['Researcher' | 'Department'| 'Interest']",
  "firstName": "String: optional",
  "lastName": "String: optional",
  "website": "String: optional",
  "edges": Array: optional
   [
    {
     "type": "String: required",
     "name": "String: required"
    }
   ]
 }
]
Import CSV - Nodes Only

Import a list of items and their metadata, but not their relations to other items. Use the following schema.

name,type,firstName,lastName,website

- Only “name” and “type” are required. All the other fields can be empty.
- "Type" should be one of the following: 'Researcher' | 'Department' | 'Interest.

Import CSV - Edges Only

Import a list of connections between items, but not their metadata. Use the following schema.

source,target,sourceType,targetType

- All fields required. ‘source’ and ‘target’ correspond to the name of a node.
- The ‘sourceType’ and ‘targetType’ should be one of the following: 'Researcher' | 'Department' | ‘Interest.
- Nodes are automatically created using the ‘source’ and ‘target’ fields.

Export

The Export section gives options to export the visible visualization you built.

Export Data

There are two options: CSV and JSON.

CSV The CSV option generates two files, one containing the nodes and metadata and the other the relationship between the nodes.

JSON The JSON option generates one file containing both the nodes and metadata and their relationship.

Export Graph

There are two options: PNG and SVG.

PNG The PNG option generates a raster file with a transparent background. All the nodes and links currently present in the visualization, including those partially or totally hidden or outside the main screen, will be in the generated image.

SVG The SVG option generates a vector-based file that can be opened in the browser or any vector-based image editor (e.g., Adobe Illustrator). The main screen container clips the file, but you can manually remove this clip to expose all the nodes and links present on the visualization when it was exporte.

The Gooey FX is removed in the PNG file, but it is embedded in the SVG file, though some applications might not display this FX correcltly.

Managing Tags

Only editors and administrators can manage tags.

Managing Tags: Adding

Click on the add_circle_outline[+ icon] at the top to open the add/edit panel.

Choose the tag’s type at the top: Researcher, Interests, and Department.

Add a name (and last name for a Researcher).

Optional: Add last name (Researcher only)

Optional: Add a website (Researcher only). Attention, the system only supports one URL address.

To add connections to other tags, start typing, and the autocomplete will show options based on the query. Keep typing to narrow down the search. You can only add connections to other existing tags. That is, you must create a tag before connecting it to another tag. Interests and Departments only support connections to Researchers; Researchers connect to both Interests and Departments (separately).

After you hit save, the tag isadded to the corresponding type on the tag list.

Managing Tags: Editing

Click on info_outline[info icon] next to the tag’s name you want to edit to open the tag information panel. Or, if the tag is currently present in the visualization, a single click on the node to open the tag information tag. Then, click on edit[pencil icon] to open the edit panel.

Make the changes you need to make.

Attention: If you decide to change tag’s type — from Researcher to Interest or Department, or from Interest or Department to Researcher — all the previous connections are removed.

Click on the clear[x icon] next to the tag to remove connections.

After you hit save, the tag is updated on the tag list, and in the visualization if current present on it.

Managing Tags: Removing

Click on info_outline[info icon] next to the tag’s name you want to edit to open the tag information panel. Or, if the tag is currently present in the visualization, a single click on the node to open the tag information tag. Then, click on edit[pencil icon] to open the edit panel.

Click on Delete.

After you confirm, the tag is permanently removed from the visualization and the tag list.

Managing User Accounts

Only administrators can manage users' accounts.

Managing Users' Accounts: User type

Guest: No account. Access to all features except export data, manage tags, and manage user accounts.

User: Access to all features except export data, manage tags, and manage user accounts.

Editor: Access to all features, including manage tags, and export data, but cannot manage user accounts.

Administrator: Full access to all features, including export data, manage tags, and manage user accounts.

Managing User Accounts: Adding

Click on the add_circle_outline[+ icon] at the top to open the add/edit user account.

Choose the user type: User, Edit, Administrator.

Add the first (required) and the last name.

Add an email (required)

A temporary password is automatically generated and save to the user profile when the account is created. Click on Generate Password If you want to reveal (and change) the temporary password, which you can manually send to the user.

An invitation will is sent to the user’s email when you hit the Save Button, and the user account is added to the users' list.

New users receive an invitation email with a link to create a new password before accessing the app.

Managing User Accounts: Editing

Click on info_outline[info icon] next to the user’s name you want to edit to open the user account information panel.

Make the changes you need to make.

Save the changes.

Managing User Accounts: Removing

Click on info_outline[info icon] next to the user’s name you want to edit to open the user account information panel.

Click on Delete.

After you confirm, the user account is permanently removed from the user account list.

Managing User Accounts: Recovering Password

If a user forgets the password, instruct them to choose the forget password option on the login page.

Fill the form with the user email.

The user will receive an email with a link to create a new password in order to access the visualization.
